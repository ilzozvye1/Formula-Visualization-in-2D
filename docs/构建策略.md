# 构建策略文档

## 📋 概述

本项目采用 **GitHub Actions + 本地下载** 的混合构建策略，实现自动化、可复现的构建流程。

---

## 🏗️ 构建架构

### 构建流程图

```
开发者提交代码
    ↓
GitHub Actions 自动触发
    ↓
在统一的 Windows 环境中构建
    ↓
上传构建产物 (Artifact)
    ↓
开发者/测试人员下载构建
    ↓
本地测试或发布
```

### 核心组件

| 组件 | 位置 | 用途 |
|------|------|------|
| **GitHub Actions** | `.github/workflows/build.yml` | 自动构建 |
| **下载脚本** | `scripts/download-builds.ps1` | 下载构建 |
| **构建产物** | `builds/` | 存储下载的构建 |
| **快速启动** | `download.bat` | 一键下载 |

## 📁 分支策略

### 分支结构

```
main (生产分支)
  ↑ 合并
develop (开发分支)
  ↑ 合并
feature/* (功能分支)
```

### 分支说明

| 分支 | 用途 | 保护级别 | 触发构建 |
|------|------|----------|----------|
| **main** | 生产环境，稳定版本 | 受保护 | ✅ 创建 Release |
| **develop** | 日常开发，集成分支 | 一般 | ✅ 临时构建 |
| **feature/*** | 新功能开发 | 无 | ❌ 可选 |

### 分支工作流程

1. **开发新功能**
   ```bash
   git checkout -b feature/new-feature develop
   # 开发...
   git push origin feature/new-feature
   ```

2. **合并到 develop**
   ```bash
   git checkout develop
   git merge feature/new-feature
   git push origin develop
   # 触发临时构建
   ```

3. **发布版本**
   ```bash
   git checkout main
   git merge develop
   git push origin main
   # 触发正式构建，创建 Release
   ```

## 🔨 构建类型

### 1. 临时构建 (Development Build)

**触发条件**:
- Push 到 `develop` 分支
- Pull Request

**特点**:
- Artifact 名称：`windows-build-dev-{commit_sha}`
- 保留时间：**7 天**
- 用途：开发测试、快速验证
- 下载方式：使用下载脚本

**GitHub Actions 配置**:
```yaml
- name: Upload artifacts (Develop/PR)
  if: github.event_name == 'pull_request' || 
      (github.ref != 'refs/heads/main' && 
       github.ref != 'refs/heads/master')
  uses: actions/upload-artifact@v4
  with:
    name: windows-build-dev-${{ steps.version.outputs.short_sha }}
    path: dist/
    retention-days: 7
```

### 2. 正式发布 (Release Build)

**触发条件**:
- Push 到 `main` 或 `master` 分支

**特点**:
- Artifact 名称：`windows-build-release-{version}`
- 保留时间：**永久**（GitHub Release）
- 用途：正式版本、用户分发
- 下载方式：GitHub Releases 页面

**GitHub Actions 配置**:
```yaml
- name: Upload artifacts (Release)
  if: github.event_name == 'push' && 
      (github.ref == 'refs/heads/main' || 
       github.ref == 'refs/heads/master')
  uses: actions/upload-artifact@v4
  with:
    name: windows-build-release-${{ steps.version.outputs.version }}
    path: dist/

- name: Create Release
  if: github.event_name == 'push' && 
      (github.ref == 'refs/heads/main' || 
       github.ref == 'refs/heads/master')
  uses: softprops/action-gh-release@v1
  with:
    tag_name: v${{ steps.version.outputs.version }}
    name: Release v${{ steps.version.outputs.version }}
    files: |
      dist/*.exe
      dist/*.zip
```

## 📥 下载策略

### 自动下载脚本

**位置**: `scripts/download-builds.ps1`

**功能**:
1. 获取最近的构建列表
2. 识别临时构建（`windows-build-dev-*`）
3. 下载最近 3 个构建
4. 自动解压到 `builds/` 目录
5. 生成构建信息文件
6. 自动清理旧构建（保留 3 个）

**使用方法**:
```powershell
# 方法 1: 使用批处理文件（推荐）
.\download.bat <GitHub_Token>

# 方法 2: 直接使用 PowerShell
powershell -File .\scripts\download-builds.ps1 -Token "ghp_xxx"

# 方法 3: 自定义参数
powershell -File .\scripts\download-builds.ps1 `
  -Token "ghp_xxx" `
  -Repo "ilzozvye1/Formula-Visualization-in-2D" `
  -Branch "develop" `
  -KeepCount 3
```

### 下载目录结构

```
builds/
├── 5_960b349/              # 构建编号_Commit 哈希
│   ├── build-info.txt      # 构建信息
│   ├── win-unpacked/       # x64 版本
│   └── win-ia32-unpacked/  # ia32 版本
├── 4_da97b52/
│   └── ...
└── 3_b9b6ff6/
    └── ...
```

### 构建信息文件

每个构建包含 `build-info.txt`:
```

Created At: 03/10/2026 09:25:48
```

## 🗑️ 清理策略

### 临时构建清理

**GitHub 端**:
- 保留时间：7 天
- 自动删除：GitHub Actions 自动清理

**本地端**:
- 保留数量：最近 3 个
- 清理方式：下载脚本自动清理
- 手动清理：运行 `cleanup-build-output.bat`

### 旧版本清理

```powershell
# 清理 build-output 目录（旧的本地构建）
.\cleanup-build-output.bat

# 手动清理 builds 目录
Remove-Item -Path ".\builds" -Recurse -Force
# 重新下载最新构建
.\download.bat ghp_xxx
```

## 🔒 安全策略

### Token 管理

**生成 Token**:
1. 访问 https://github.com/settings/tokens
2. 创建新 Token (classic)
3. 勾选权限：`repo` (全部)
4. 生成并复制

**Token 使用**:
- ⚠️ **不要**提交到 Git
- ⚠️ **不要**在公共场合分享
- ✅ 使用 `.env` 文件存储（已添加到 .gitignore）
- ✅ 定期轮换

**示例 .env 文件**:
```bash
# scripts/.env (不要提交到 Git)
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx
```

### 构建安全

- ✅ 使用 GitHub 官方 Actions
- ✅ 固定 Action 版本（如 `@v4`）
- ✅ 最小权限原则
- ✅ 自动清理敏感信息

## 📝 故障排除

### 常见问题

**Q1: 下载失败 "401 Unauthorized"**
- **原因**: Token 无效或过期
- **解决**: 重新生成 Token

**Q2: 找不到 artifacts**
- **原因**: 构建超过 7 天被删除
- **解决**: 触发新构建或从 Releases 下载

**Q3: 文件被占用无法删除**
- **原因**: 构建程序正在运行
- **解决**: 关闭程序后重试

**Q4: 构建太慢**
- **原因**: 网络或 GitHub 服务器
- **解决**: 等待或重试
