## 二、核心指标体系

### 2.1 商业指标

| 指标 | 英文 | 定义 | 目标 |
|------|------|------|------|
| 收入 | Revenue | 总付费金额 | 月增长20% |
| 付费用户数 | Paying Users | 活跃Pro用户数 | 1000+ |
| 付费转化率 | Conversion Rate | 付费/总用户 | ≥3% |
| 客单价 | ARPU | 收入/付费用户 | ≥¥50 |
| 终身价值 | LTV | 用户贡献总收入 | ≥¥100 |
| 获客成本 | CAC | 营销费用/新用户 | ≤¥30 |
| LTV/CAC比 | LTV:CAC | LTV除以CAC | ≥3 |

### 2.2 用户指标

| 指标 | 英文 | 定义 | 目标 |
|------|------|------|------|
| 日活用户 | DAU | 每日活跃用户 | 1000+ |
| 月活用户 | MAU | 每月活跃用户 | 10000+ |
| 用户留存率 | Retention | 次日/7日/30日留存 | 40%/20%/10% |
| 用户流失率 | Churn Rate | 月流失用户/总用户 | ≤5% |
| 活跃天数 | Active Days | 用户月活跃天数 | ≥5天 |

### 2.3 产品指标

| 指标 | 英文 | 定义 | 目标 |
|------|------|------|------|
| 功能使用率 | Feature Usage | 功能使用次数/DAU | 持续监控 |
| 导出次数 | Export Count | 导出功能使用次数 | 提升30% |
| 预设使用率 | Preset Usage | 预设公式使用占比 | 提升20% |
| 页面停留时间 | Session Duration | 用户平均使用时长 | ≥5分钟 |
| 错误率 | Error Rate | 功能异常/调用次数 | <1% |

## 三、用户行为分析

### 3.1 用户旅程

```
用户旅程阶段
│
├── 认知期（Awareness）
│   ├── 来源渠道
│   ├── 首次访问
│   └── 落地页表现
│
├── 转化期（Conversion）
│   ├── 注册行为
│   ├── 首次使用
│   └── 付费转化
│
├── 留存期（Retention）
│   ├── 再次访问
│   ├── 功能探索
│   └── Pro续费
│
└── 推荐期（Referral）
    ├── 口碑传播
    └── 社交分享
```

### 3.2 关键行为节点

| 阶段 | 关键行为 | 数据采集 |
|------|----------|----------|
| 注册 | 完成注册 | 注册时间、渠道 |
| 首次使用 | 绘制第一个函数 | 函数类型、耗时 |
| 付费触发 | 点击升级按钮 | 触发场景、页面 |
| 续费 | Pro到期前行为 | 续费意愿信号 |

### 3.3 用户分群

| 分群 | 定义 | 运营策略 |
|------|------|----------|
| 潜在用户 | 注册但未付费 | 引导试用Pro |
| 试用用户 | 刚升级Pro | 引导深度使用 |
| 活跃Pro | 高频使用 | 续费提醒 |
| 流失风险 | 30天未访问 | 召回策略 |
| 沉默用户 | 注册未使用 | 激活引导 |

## 四、数据采集设计

### 4.1 采集事件

```javascript
// 用户行为事件
const EVENTS = {
  // 账户相关
  'user.register': { userId, source },
  'user.login': { userId, method },
  'user.logout': { userId, duration },
  
  // 功能相关
  'function.add': { functionType, category },
  'function.delete': { functionId },
  'function.edit': { changes },
  'preset.use': { presetId, category },
  'export.start': { format, hasPro },
  'export.complete': { format, resolution },
  
  // 付费相关
  'upgrade.click': { source, page },
  'upgrade.success': { plan, amount },
  'license.activate': { keyType, deviceId },
  
  // 错误相关
  'error.render': { functionId, errorType },
  'error.export': { format, errorMsg }
};
```

### 4.2 数据埋点

| 场景 | 埋点位置 | 采集数据 |
|------|----------|----------|
| 页面加载 | 入口文件 | 页面URL、来源、用户ID |
| 功能使用 | 函数输入框 | 函数表达式、渲染时间 |
| 付费点击 | 升级按钮 | 按钮位置、用户类型 |
| 错误发生 | 异常处理 | 错误类型、堆栈信息 |

## 六、数据驱动决策

### 6.1 决策场景

| 场景 | 数据依据 | 决策 |
|------|----------|------|
| 定价调整 | 付费转化率、竞品价格 | 调价幅度 |

| 引导时机 | 3个函数后vs5个后 | 转化率 |
