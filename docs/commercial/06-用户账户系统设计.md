## 二、账户类型设计

### 2.1 账户层级

```
用户账户体系
│
├── 游客（未登录）
│   ├── 本地存储数据
│   ├── 体验基础功能
│   └── 无法同步
│
├── 免费账户（注册用户）
│   ├── 云端存储数据
│   ├── 基础功能免费
│   └── 可升级Pro
│
└── Pro账户（付费用户）
    ├── 全部Pro功能
    ├── 优先客服
    └── 专属预设
```

## 三、数据同步设计

### 3.1 同步数据范围

| 数据类型 | 本地存储 | 云端同步 | 说明 |
|----------|----------|----------|------|
| 用户账户 | ❌ | ✅ | 账户信息 |
| Pro状态 | ✅ | ✅ | 许可证状态 |
| 预设公式 | ✅ | ✅ | 用户自定义 |
| 画布配置 | ✅ | ✅ | 方程和样式 |
| 导出历史 | ❌ | ❌ | 本地记录 |

### 3.2 同步机制

```javascript
// 跨设备同步
class DataSync {
  // 同步触发条件
  static SYNC_TRIGGERS = [
    '登录时',           // 登录后首次同步
    '打开应用时',       // 检查云端更新
    '修改数据时',       // 本地变更同步
    '每5分钟',         // 定时同步
    '切换网络'         // 网络状态变化
  ];
  
  // 冲突解决策略
  static RESOLUTION = {
    newer: '保留最新',    // 时间戳比较
    cloud: '优先云端',    // 重要数据
    local: '优先本地',    // 用户输入
    manual: '用户选择'    // 冲突时手动
  };
}
```

## 四、账户安全设计

### 4.1 安全机制

| 机制 | 说明 | 实现 |
|------|------|------|
| 密码加密 | 哈希+盐值存储 | bcrypt |
| 登录验证 | JWT Token | 7天有效期 |
| 设备限制 | 同账号最多3设备 | 设备指纹 |
| 异常检测 | 异地登录提醒 | IP检测 |

### 4.2 账户安全流程

```
用户登录
    ↓
验证账号密码
    ↓
生成JWT Token
    ↓
检查设备指纹
    ↓
┌─ 新设备 → 发送验证邮件
│
└─ 验证通过 → 登录成功
```
